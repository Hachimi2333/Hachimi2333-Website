# 用户认证系统重组设计

> [!NOTE]
> This document may not reflect the current implementation.
> See the final report for up-to-date state:
> [Final Report](../reports/user-auth-restructure.md)

## [S1] 问题

当前用户系统与网站其他部分混合在一起，登录状态显示在导航栏中，用户体验不够清晰。需要将用户系统独立出来，放在 `/auth` 路由下，并提供专门的用户首页。

## [S2] 解决方案概述

1. 创建 `/auth` 首页，显示用户资料概览，未登录用户显示提示信息
2. 简化导航栏，只保留主题切换按钮
3. 删除原 `/auth/profile` 路由，将功能整合到 `/auth` 首页

## [S3] 详细设计

### 1. `/auth` 首页设计

**路由:** `/auth`
**组件:** `AuthHomeView.vue`

**功能:**
- 已登录用户：显示用户资料概览（头像、名称、邮箱等）
- 未登录用户：显示提示信息和登录/注册按钮

**页面布局:**
```
┌─────────────────────────────┐
│         Auth Home           │
├─────────────────────────────┤
│  [已登录状态]               │
│  ┌─────────────────────┐   │
│  │  用户头像            │   │
│  │  用户名称            │   │
│  │  邮箱                │   │
│  │  [编辑资料] [退出]   │   │
│  └─────────────────────┘   │
│                             │
│  [未登录状态]               │
│  ┌─────────────────────┐   │
│  │  提示信息            │   │
│  │  [登录] [注册]       │   │
│  └─────────────────────┘   │
└─────────────────────────────┘
```

### 2. 导航栏简化

**修改组件:** `AppHeader.vue`

**变更:**
- 移除用户头像/图标
- 移除登录按钮
- 移除用户下拉菜单
- 只保留主题切换按钮

**最终导航栏:**
```
┌─────────────────────────────┐
│  [Logo]    [首页]  [主题]   │
└─────────────────────────────┘
```

### 3. 路由调整

**删除:**
- `/auth/profile` 路由

**修改:**
- `/auth` 路由指向新的 `AuthHomeView.vue`

**保留:**
- `/auth/login` 路由
- `/auth/register` 路由

### 4. 组件迁移

**从 `ProfileView.vue` 迁移:**
- 用户信息显示逻辑
- 编辑功能（如果有）
- 退出登录功能

**到 `AuthHomeView.vue`:**
- 整合用户信息显示
- 添加未登录状态处理

## [S4] 技术实现要点

1. **状态管理:** 继续使用 `useAuth()` composable
2. **路由守卫:** 调整路由守卫逻辑，移除对 `/auth/profile` 的特殊处理
3. **样式:** 使用现有的 shadcn-vue 组件和 Tailwind CSS
4. **响应式:** 确保在移动端和桌面端都有良好的显示效果

## [S5] 测试要点

1. **功能测试:**
   - 未登录用户访问 `/auth` 显示提示信息
   - 已登录用户访问 `/auth` 显示用户资料
   - 导航栏只显示主题切换按钮
   - 登录/注册功能正常工作

2. **路由测试:**
   - `/auth` 路由正确指向 `AuthHomeView.vue`
   - `/auth/profile` 路由已删除
   - 登录后跳转逻辑正常

3. **UI 测试:**
   - 导航栏在不同屏幕尺寸下正常显示
   - 用户资料页面布局合理
   - 未登录提示信息清晰可见
